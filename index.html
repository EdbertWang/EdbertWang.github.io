Weekly Notes

  • Digital Information: Data stored on computers or similar computer readable forms
  • Reproducible, Easily Shareable, Flexible, Modifiable, Hard to intercept
  • Quantity - Commonly found in very large datasets, everything gathers data on users, and more things are integrating with the web

  • Computers are data processing devices
    • Input, process, store and output data
    • Hardware
      • External - I/O Devices: Mouse, Monitor, Printer, Microphone
      • Internal - Process and store data: Drives, Processors, RAM

  • Computers Networks
    • Allow for resource sharing and communication, but adds complexity and makes security more difficult
    • Client to server Networks
      • Fixed server computer provides services, and clients access said server
      • Centralized system offers faster and more secure data
      • High maintainance costs, and high inital investment
    • Peer to Peer Networks
      • Fluid network of roughly equal computers sharing resources
      • Decentralized, low cost
      • Security and version control become more difficult
    • The Internet
      • Network of networks connecting billions of computers, uses TCP/IP
      • TCP/IP
        • Application: HTTP, Establishes server connections
        • Transport - TCP, splits request into packets and a checksum
        • Internet - IP, packets sent over routers
        • Network - Network Cards, Hardware conberts packet to binary

  • Web Usability - Measure of how well a user can use a product/design to achieve a goal
    • Learnable: First time users should be able to quickly complete basic tasks
    • Efficient: Tasks should be quicklu completeable
    • Memorable: Users should be albet to quickly reorient themselves to the website
    • Forgiving: Errors should be quickly resolvable, and shouldn't occur too often
    • Satisfying: Users should enjoy using the system
  • User Experience Design - Process of designing a website for the user
    1. Research - Understand what users need and want
    2. Design - Create and interface matching the needs
    3. Evaluate - Get feedback on designs
  • Usability Testing
    • Facilitator gives participant tasks on website who then gives feedback on their Experience
    • Factors to consider: Participant background, Facilitator interference, access method/location

  • CSS
    • Cascading Style Sheets - define how html elements are displayed
    • Used for global definitions, reduces load times, offers more customizability, and maintains concistancy throuhgout a series of conected pages
    • Definitions concist of a selector, followed by properties that you want to change, and the values you want to change them too
    • Can be attached inline to change a specific line, interally to change a specific page, or externally for use over many pages
      • Precedence: Inline > Internal > External > Browser default
    • Common use for CSS is to order sections, or to define spacing for elements
      • Ex. In HTML: class = "left" In CSS: .class{float: left;}
      • Padding, Margin, and Border can all be used to increase spacing

  • Accessibility - Making a website useable by as many people as possible
    • Perceivable - Content should be percieveable to all senses, including augmented ones : Color shouldn't be used to convey information
    • Understandable - content should be clearly written and organized : Labels or instructions should be provided for input sections
    • Robust - Webpage should be accessible from a wide range of devices : Write good code, have mobile pages
    • Operable - Users should be able to operate all controls such as buttons or input forms : Functionality shouldn't be dependant on input timings
    • Disabilites - Guide
      • Visual - Color/Full Blindness
      • Hearing - Deafness/Hard of Hearing
      • Mobility - Motor control issues, slowness
      • Cognitive - Learrning or memory disabilites
    • Do the following
      • Good text/background contrast
      • Alternative reading - Screen Readers, Captioning
      • Space out important elements
      • Reduce outside dependancies - Ex. Flash
